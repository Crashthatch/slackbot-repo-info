Health Check for
Github Libraries

Identify strengths and weaknesses of public Github libraries without leaving Slack.
Spot red flags before adding a dependency you'll regret.

1 - Stars, Watchers and Forks

Repositories with more users are likely to have fewer bugs thanks to more extensive testing.
A low count here could be a warning sign.

2 - Recent Activity

We show repo age & last push time. If a library isn't actively developed, bug-fixes and security patches may be slow.

3 - Dependencies Checking

We warn when a library's dependencies are outdated, meaning poorer compatibility, and maybe security issues.
22 package managers supported thanks to libraries.io

Get Started

Add to Slack, then invite the @repo_info bot to the channels you want to monitor for github links.

/invite @repo_info #general
@repo_info bot will respond whenever someone links to a github repository.

About

Repo Info was created for the AWS Serverless Slackbot Competition by @thomasfletcher. It uses Claudia.js and AWS Lambda to provide a slack bot that listens to Slack Events and operates scalably without servers.

Data comes from the Github API and Libraries.io API.
