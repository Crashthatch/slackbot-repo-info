Privacy Policy

We store your team's ID & name, the installing user's ID, and a couple of access tokens so that we can correctly listen for mentions of github repositories and make sure that the bot can post a message back to the right channel.

The full details stored look something like:

{
  "access_token": "xoxp-0123456789-xxxxxxxxxxx-1234567890-xxxxxxxxx",
  "bot": {
    "bot_access_token": "xoxp-0123456789-xxxxxxxxxxx-1234567890-xxxxxxxxx",
    "bot_user_id": "U29NVXXXX"
  },
  "ok": true,
  "scope": "identify,bot",
  "team_id": "T06QXXX99",
  "team_name": "Tom's Test",
  "user_id": "U06QX9XXX"
}

Messages received from Slack are received, scanned for which github repository is being immediately discarded and not stored .

Only channels which you have invited the @slack_repo bot to are monitored (The bot can only see what it's been invited to see). Slack doesn't even send us information about other channels.

We do not send any team/user specific information to anyone else. We use various APIs (Github, Libraries.io, etc.) to gather information about github repos, but the only information we need to give them is the name of the repo we want info about.

In the case of an internal error, we may store the github repo URL & the slack message that caused the error so that we can investigate.

To be honest, it would be more hassle and cost for us to read and store your messages, so it's easier (& securer) for us to just get rid of them as soon as we can (typically, messages don't ever get stored on a hard-disk, so they go away once the lambda function finishes processing and is terminated).

The Repo Info bot is Open Source if you would like to check for yourself.
